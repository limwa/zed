## Summary

- Fix Linux dead-key shortcut handling so consumed shortcuts like `Ctrl+~` on Portuguese layouts no longer leave compose state active or insert a stray `~`.
- Delete marked dead-key text instead of only unmarking it when a shortcut consumes the key event, and suppress stale Wayland IME follow-up events from ibus.
- Add regression coverage in both `editor` and `gpui_linux` for marked-text deletion semantics and consumed dead-key shortcut cancellation.
